Vacan v1 MVP

Editor visual minimo inspirado en Canva

Esta primera version prioriza una experiencia usable: subir imagenes, reutilizarlas desde un banco, crear paginas apiladas, mover elementos libremente y transformar assets con controladores sobre el canvas.

React + Vite Konva canvas shadcn/ui Vitest

Que se puede hacer

Banco de imagenes

Sube una o varias imagenes. Cada asset queda disponible en el panel izquierdo para reusarlo.

Paginas verticales

Crea multiples paginas. Se muestran una debajo de otra con conteo de capas y pagina activa.

Elementos movibles

Imagenes, textos y formas se arrastran libremente en cada canvas y muestran controladores al click.

Texto y fuentes

Agrega texto, edita el contenido, cambia tamano y selecciona entre varias familias tipograficas.

Formas basicas

Inserta cuadrados, circulos y triangulos con colores editables desde el inspector.

Exportacion

Exporta la pagina activa como PNG con pixel ratio alto para una salida nitida.

Shortcuts incluidos

  • Alt + arrastrar un elemento deja una copia en la posicion original.
  • Cmd / Ctrl + D duplica el elemento seleccionado.
  • T agrega un texto en la pagina activa.
  • Delete / Backspace elimina el elemento seleccionado.

Narrativa visual

La interfaz usa una estructura similar a Canva: barra superior compacta, panel izquierdo de herramientas y assets, espacio central con paginas apiladas, e inspector derecho para propiedades. Los controles reutilizan los componentes shadcn disponibles en el proyecto y los iconos vienen de lucide-react.

Limites de v1

  • El documento vive en estado local del navegador; Convex queda listo en el repo, pero no conectado al flujo.
  • No hay edicion rich-text dentro del canvas; el texto se edita desde el inspector.
  • No hay plantillas, agrupacion, guias inteligentes, historial undo/redo ni colaboracion en tiempo real.
  • La exportacion actual es PNG de la pagina activa.

Calidad

La logica central de documento esta separada en src/editor/document.ts y cubierta por tests unitarios en src/editor/document.test.ts: paginas, creacion de elementos, duplicacion, escalado de imagenes, updates y borrado.
